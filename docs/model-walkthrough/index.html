DiffusionGemma 26B_A4B — SFT on Sudoku: model walkthrough

Ramp-up documentation for the training/inference pipeline defined by gemma/diffusion/hackable_diffusion_adapter/configs/sft_sudoku_full.py in google-deepmind/gemma. Every tensor shape below is either traced to a file:line or measured by the smoke run in tools/rampup/smoke.py (tiny config: 3 layers, D=32, V=64; outputs in tools/rampup/measured/*.json). Symbolic dimensions are defined in the symbol legend.

0. Status & open questions

Verified by execution: one full training step (forward, both losses, backward, the exact optax chain from the config) and one full AR-diffusion sampling call (2 canvases × 4 denoising steps) ran on a tiny-but-topologically-identical model (local-sliding + global attention, MoE + dense shared FFN, self-conditioning). All shapes in the diagrams and narratives match the measured JSON. The full 26B_A4B parameter inventory was measured with jax.eval_shape (no memory needed): 25,250,986,782 params (≈25.25B, consistent with the "26B" name).
  • Diffusion time t is never fed to the network. WrappedDiffusionGemmaNetwork.__call__ accepts time but never uses it (hd_gemma_network.py:186-245). No AdaLN / time embedding exists. Noise-level information reaches the model only through the corrupted tokens and the self-conditioning logits. Verified by code reading, not by an ablation.
  • keep_last_prefill_kv=True (diffusion/_models.py:31): its downstream consumer was not traced. [UNVERIFIED]
  • "A4B" ≈ 4B active params per token (top-8 of 128 experts + dense shared MLP) is an interpretation of the class docstring; the code states no numeric active count. [UNVERIFIED]
  • self_cond_prob=0.5 is a hidden default (sft_model.py:270), not surfaced in the config file.
  • The smoke run replicated the kauldron train step (loss formulas + optax chain) manually; the kauldron TrainStep class itself and checkpoint loading (GemmaDiffusionCheckpointLoader) were not executed (no checkpoint available).
  • The smoke run used num_canvases=2 to exercise the multi-canvas machinery; the actual config uses num_canvases=1, so the AR outer loop runs exactly once in production.
  • gemma/diffusion/_sampler.py and _early_stopping.py are a parallel, unused sampler implementation — the eval path uses hackable_diffusion's hd.sampling.* instead.
  • The diffusion loss covers all valid tokens of the selected canvas (clean and corrupted alike, mask_key="target_mask"), not just corrupted positions — a deliberate-looking design choice worth knowing.

1. What this model is

DiffusionGemma_26B_A4B (gemma/diffusion/_models.py:21) is the Gemma 4 26B mixture-of-experts transformer (gemma/gm/nn/gemma4/_gemma4.py:238) reused as a discrete text-diffusion denoiser. The backbone is architecturally unchanged (30 blocks: 25 local-sliding-window + 5 global attention layers; each block has a dense shared FFN plus a 128-expert top-8 MoE FFN; weight-tied embedding/output head). Two things make it "diffusion":

  1. Externally supplied attention masks — within the canvas being denoised the mask is bidirectional; across canvases and to the prompt it is block-causal (mask_helpers.py:247-309).
  2. Self-conditioning — a small extra module (SelfConditioning, gemma/diffusion/_transformer.py:53) mixes the previous denoising step's logits (re-embedded through the tied table) into the input embeddings.

Training (SFT on Sudoku) couples two objectives on the same backbone: a causal "encoder" pass over prompt ++ clean-answer trained with ordinary next-token CE, whose KV cache then conditions a denoiser pass that reconstructs the clean answer tokens from a uniformly-corrupted copy (x0-prediction cross-entropy). Inference generates the answer canvas-by-canvas autoregressively; each canvas is filled by an inner diffusion loop (32/64/96 steps of D3PM-style stay/renoise/clean routing), then appended to the KV cache.

4.1 Inside sft_encode (encoder prefill)

Expansion of the "sft_encode" node above (sft_model.py:159-217; hd_gemma_network.py:52-107). One standard causal Gemma forward over prompt ++ x0 [B, FS] — the same layer_i/*, embedder and final_norm weights as the denoiser passes, but via Transformer.__call__, so the self_conditioner module is not run. It produces both training signals at once: encoder_logits [B, FS, V] for the AR loss, and the prefilled KV cache whose per-example write cursor is then rewound to P + selected_idx·CS so the denoiser reads only prompt + prior canvases as context. In the explorer, this diagram is a drill-down of the sft_encode node (Training forward → sft_encode).

5. Inference sequence (AR diffusion sampling)

Both loops are explicit: the outer autoregressive canvas loop (hackable_diffusion .../ar_diffusion_sampler.py:244-332) and the inner denoising loop (.../sampling.py:190-281). With the production config NC = max_num_canvases = 1, the outer loop body runs once. The early-stopping evaluator variant wraps the inner loop in a while that exits when mean token entropy ≤ 0.05 (.../diffusion_early_stopping.py:117-134).

9. Life of a Batch (training)

  1. Read + format. A Bagz record {puzzle, solution} is parsed (sudoku_data.py:95-107), the puzzle wrapped in the chat template (sft_sudoku_full.py:130-136) and both sides tokenized with the Gemma4 tokenizer (sudoku_data.py:177-178).
  2. Pad + chunk. Prompt right-padded to [P]=[256] (sudoku_data.py:189). CanvasChunker (data.py:64-102) writes the response into an NC·CS canvas buffer: EOS-fills the last partially-used canvas, PAD-fills unused ones, and emits canvas [TC], canvas_id [TC] (0-based segment index per token), canvas_mask [TC] (True on valid canvases). SequenceTargetShift (data.py:121-152) rolls prompt ++ canvas left by one to make AR targets: encoder_target [FS], encoder_target_mask [FS] (True only where current and next position are valid). After batching + a trailing channel dim on canvas: prompt int32[B,P], canvas int32[B,TC,1], canvas_id[B,TC], canvas_mask bool[B,TC], encoder_target[B,FS], encoder_target_mask[B,FS] (measured: [2,8], [2,16,1], [2,16], [2,16], [2,24], [2,24]).
  3. Sample time, corrupt. t ~ U(1e-4, 1−1e-4), shape [B,1,1] (sft_model.py:307; hd time_sampling.py:121-125). Each canvas token independently keeps its value w.p. α(t)=1−t or is replaced by a uniform random token: xt [B,TC,1] (sft_model.py:310; hd discrete.py:277-290). This is a uniform categorical process — there is no [MASK] token.
  4. Pick a canvas. selected_canvas_idx ~ U[0, num_valid_canvases) [B] (sft_model.py:321-334). With NC=1 it is always 0.
  5. Encoder prefill (causal AR pass). full_seq = prompt ++ x0_tokens [B,FS]; positions = cumsum(valid)−1; causal prefill mask [B,FS,FS] (sft_model.py:159-217, hd_gemma_network.py:52-107, mask_helpers.py:69-140). One standard Gemma forward returns encoder_logits [B,FS,V] (measured bf16[2,24,64]) and fills the KV cache (per layer: local k,v [B,C,N_kv,d_h], global k,v [B,C,N_gkv,K_g] (measured [2,24,2,8] / [2,24,1,16])). Then end_index := P + idx·CS (sft_model.py:214-215, mask_helpers.py:317-344) so the denoiser treats prompt + prior canvases as its causal context. Gradients flow through the cache (stop_gradient…=False, sft_model.py:355).
  6. Denoiser pass 1. Attention mask [B,TC,C] (mask_helpers.py:247-309): each canvas query sees non-PAD prompt KV and KV of canvases with id ≤ selected — bidirectional inside the selected canvas, future canvases hidden. The wrapped network embeds xt, applies self-conditioning with zero sc_logits, runs the 30 blocks against the cache, decodes through the tied table + softcap: logits [B,TC,V] (hd_gemma_network.py:186-245, diffusion/_transformer.py:86-184) (measured mask bool[2,16,24], logits bf16[2,16,64]).
  7. Self-conditioning + denoiser pass 2. Pass-1 logits are stop-gradiented and kept per-example w.p. 0.5, else zeroed (sft_model.py:397-411). A second identical decode with sc_logits produces the loss-bearing logits (sft_model.py:413).
  8. Losses. target_mask = canvas_mask & (canvas_id == idx) [B,TC] (sft_model.py:380). diffusion_loss: plain CE(pass-2 logits, clean x0) over target_mask positions, mean per example → [B], no SNR weighting (hd discrete_loss.py:41-122). encoder_loss: masked next-token CE(encoder_logits, encoder_target) → [B] (sft_model.py:608-643). Total = 1.0·mean + 1.0·mean (measured: 4.09/4.09 + 4.19/4.16 → 8.25 ≈ 2·ln 64, as expected for an untrained tiny model).
  9. Backward + update. jax.grad w.r.t. params (kauldron train_step.py:292-301), then the optax chain in config order: global-norm clip 1.0 → factored RMS (Adafactor) → weight decay 1e-4 → warmup-cosine LR (sft_sudoku_full.py:190-205); params are FSDP-sharded. (Smoke run executed this exact chain; shapes preserved.)

10. Life of a Sample (inference)

  1. Entry. eval_main loads a checkpoint and runs an evaluator (eval_main.py:126-207) → GemmaSamplingEvaluator._ar_diffusion_step (sft_model.py:506-549) builds cond = {prompt_tokens [B,P], prompt_lengths [B]} and calls the AR sampler.
  2. Prompt prefill. init_ar_state (hd_gemma_ar_state_handler.py:170-270) allocates the cache with C = P + NC·CS, runs one causal encoder pass over the prompt (cache end_index = P), and prepares canvas positions prompt_lengths + arange(CS) [B,CS] and the decoder mask [B,CS,C] (measured [2,8], [2,8,24]).
  3. Outer AR loop (ar_diffusion_sampler.py:244-332), while canvases remain and not all done:
    1. Init noise. x_T ~ Uniform(V) [B,CS,1] — uniform random tokens, not masks (hd discrete.py:216-228).
    2. Inner diffusion loop (hd sampling.py:190-281), T_s steps with t descending ~1 → ~0 (time_scheduling.py:96-121). Each step: inject previous logits as sc_logits (hd_gemma_ar_state_handler.py:46-64); call the network → logits [B,CS,V]; DiscreteDDIMStep.update (hd discrete_step_sampler.py:844-929) samples x0_hat ~ categorical(logits/0.7) and routes each token among stay (keep x_t) / renoise (fresh uniform) / clean (take x0_hat) with posterior weights from α=1−t. Output x_(t−1) [B,CS,1].
    3. Commit canvas. update_ar_state (hd_gemma_ar_state_handler.py:276-359): truncate at stop tokens (EOS / END_OF_TURN / BEGIN_OF_TOOL_RESPONSE, ar_eval.py:100-104) → done |= stopped; one forward pass appends the canvas K/V to the cache (end_index += CS); write tokens into predicted_tokens [B,C]; advance positions and rebuild the mask for the next canvas.
  4. Finalize. Return predicted_tokens[:, P:] [B, NC·CS] (hd_gemma_ar_state_handler.py:365-380) (measured int32[2,16]; 2 canvases processed, 3 recorded denoising steps each). Sudoku metrics run host-side on the detokenized samples (sudoku_eval.py).

Train vs inference network call: same backbone method; differences are only in orchestration — cache prefilled from prompt ++ x0 (train) vs prompt-only then appended per canvas (inference); queries = all canvases [B,TC,C] (train) vs one canvas [B,CS,C] (inference); self-conditioning two-pass w.p. 0.5 (train) vs every step from the previous iteration (inference); is_training flag.

11. How this was verified

tools/rampup/smoke.py builds the tiny model (3 layers = 2 local + 1 global, D=32, V=64, MoE 4 experts/top-2 — same topology, every dim shrunk), pushes dummy examples through the real data transforms (CanvasChunker, SequenceTargetShift), runs one full training step and one full AR sampling call, and records every flax module call's input/output shapes via nn.intercept_methods. Outputs:

  • tools/rampup/measured/module_io.json — 1,011 recorded module calls (init / train_forward / inference phases)
  • tools/rampup/measured/weights_tiny.json — tiny inventory, 50,959 params
  • tools/rampup/measured/weights_full_26b.json — full 26B_A4B inventory via jax.eval_shape, 25,250,986,782 params
  • tools/rampup/measured/summary.json — batch/preds/final-state shapes, loss values

No shape in this document was copied from a docstring without being confirmed by either the measured JSON or the constructing code line.
